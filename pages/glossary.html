Kernel și Componente Core

Nucleul sistemului de operare și componentele fundamentale
▼
🔧 Kernel (Nucleu)
De bază

Componenta centrală și cea mai importantă a unui sistem de operare. Kernel-ul acționează ca un pod între aplicații și hardware, gestionând resursele sistemului și oferind servicii esențiale tuturor programelor.

🎯 Funcții principale:

  • Gestionarea proceselor - crearea, planificarea și terminarea proceselor
  • Gestionarea memoriei - alocarea și eliberarea memoriei RAM
  • Gestionarea dispozitivelor - comunicarea cu hardware prin drivere
  • Gestionarea fișierelor - operații de citire/scriere pe disc
  • Apeluri de sistem - interfața dintre user space și kernel space

📊 Tipuri de kernel:

  • Monolitic - tot codul în kernel space (Linux, Windows NT)
  • Microkernel - minimal, servicii în user space (Minix, QNX)
  • Hibrid - combinație (macOS XNU, Windows modern)
  • Exokernel - acces direct la hardware (experimental)
# Linux - vizualizare informații kernel $ uname -r 5.15.0-generic # Windows - versiune kernel > ver Microsoft Windows [Version 10.0.22621]
Driver Proces System Call
📞 System Call (Apel de sistem)
Avansat

Mecanismul prin care un program din user space solicită un serviciu de la kernel. System calls sunt poarta de acces controlată către resursele hardware și funcțiile privilegiate ale SO.

🔄 Cum funcționează:

  1. Programul apelează o funcție wrapper (ex: open())
  2. Se execută instrucțiunea syscall sau int 0x80
  3. CPU comută în kernel mode
  4. Kernel-ul execută operația cerută
  5. Rezultatul este returnat programului
// C - exemplu system call pentru citire fișier #include <unistd.h> #include <fcntl.h> int fd = open("file.txt", O_RDONLY); // syscall: open char buffer[100]; read(fd, buffer, 100); // syscall: read close(fd); // syscall: close
Kernel Privilegii API
⚙️ Driver (Șofer de dispozitiv)
De bază

Software specializat care permite sistemului de operare să comunice cu un dispozitiv hardware specific. Driverele traduc comenzile generice ale SO în instrucțiuni specifice dispozitivului.

📦 Tipuri de drivere:

  • Character device - flux de date secvențial (tastatură, mouse)
  • Block device - acces în blocuri (hard disk, SSD)
  • Network device - interfețe de rețea
  • Virtual device - dispozitive software (null, random)
# Linux - listare drivere încărcate $ lsmod Module Size Used by nvidia 51445760 0 snd_hda_intel 57344 0 # Windows PowerShell > Get-WmiObject Win32_PnPSignedDriver | Select DeviceName, DriverVersion
Kernel Interrupt I/O
🔗 Bootloader
Avansat

Primul program care rulează la pornirea calculatorului. Bootloader-ul inițializează hardware-ul, încarcă kernelul în memorie și îi transferă controlul.

🔄 Procesul de boot:

  1. POST - Power-On Self Test (BIOS/UEFI)
  2. MBR/GPT - citire tabel partiții
  3. Stage 1 - bootloader primar (mic, în MBR)
  4. Stage 2 - bootloader complet (GRUB, Windows Boot Manager)
  5. Kernel - încărcare și execuție
  6. Init - primul proces user space
BIOS UEFI Kernel
💻 BIOS (Basic Input/Output System)
De bază

Firmware tradițional stocat pe un chip de pe placa de bază, care inițializează hardware-ul și pornește procesul de boot. A fost înlocuit treptat de UEFI.

⚡ Limitări BIOS:

  • Mod real 16-bit
  • Max 2TB per disc (MBR)
  • Max 4 partiții primare
  • Interfață text simplă
  • Fără suport rețea nativ
UEFI Bootloader ROM
🚀 UEFI (Unified Extensible Firmware Interface)
Avansat

Succesorul modern al BIOS-ului, oferind o interfață grafică, boot mai rapid, suport pentru discuri mari și caracteristici de securitate precum Secure Boot.

✨ Avantaje UEFI:

  • Mod protejat 32/64-bit
  • Suport GPT - discuri > 2TB
  • 128 partiții per disc
  • Interfață grafică cu mouse
  • Secure Boot - protecție malware
  • Boot de pe rețea (PXE îmbunătățit)

Procese și Thread-uri

Execuția programelor și managementul sarcinilor
▼
📦 Proces
De bază

O instanță a unui program în execuție. Fiecare proces are propriul spațiu de adrese, resurse alocate și cel puțin un thread de execuție. Procesele sunt izolate între ele pentru securitate și stabilitate.

📊 Stările unui proces:

  • New (Nou) - procesul este creat
  • Ready (Pregătit) - așteaptă să fie executat
  • Running (Rulează) - instrucțiunile sunt executate
  • Waiting (Așteaptă) - așteaptă un eveniment I/O
  • Terminated (Terminat) - execuția s-a încheiat
# Linux - vizualizare procese $ ps aux USER PID %CPU %MEM VSZ RSS TTY STAT START TIME COMMAND root 1 0.0 0.1 168936 11520 ? Ss 10:00 0:02 /sbin/init # Windows - procese > Get-Process | Sort-Object CPU -Descending | Select -First 10
Thread Scheduler Context Switching
🔄 Thread (Fir de execuție)
De bază

Cea mai mică unitate de execuție care poate fi planificată de SO. Thread-urile din același proces împart același spațiu de adrese și resurse, dar au propria stivă și registre.

⚖️ Proces vs Thread:

  • Memorie: Proces = separată, Thread = partajată
  • Creare: Proces = lentă, Thread = rapidă
  • Comunicare: Proces = IPC complex, Thread = directă
  • Izolare: Proces = completă, Thread = parțială
# Python - exemplu thread-uri import threading def task(name): print(f"Thread {name} rulează") t1 = threading.Thread(target=task, args=("A",)) t2 = threading.Thread(target=task, args=("B",)) t1.start() t2.start()
Proces Mutex Deadlock
⏱️ Scheduler (Planificator)
Avansat

Componenta kernel-ului care decide care proces/thread primește timp de CPU și pentru cât timp. Obiectivul este maximizarea utilizării CPU și minimizarea timpului de așteptare.

📋 Algoritmi de planificare:

  • FCFS (First Come First Served) - simplu, dar poate cauza "convoy effect"
  • SJF (Shortest Job First) - optimal, dar dificil de prezis
  • Round Robin - time slices egale, echitabil
  • Priority Scheduling - bazat pe priorități
  • CFS (Completely Fair Scheduler) - Linux modern
Proces Context Switching
💀 Deadlock (Blocare reciprocă)
Avansat

Situație în care două sau mai multe procese se blochează permanent, fiecare așteptând resurse deținute de celelalte. Niciun proces nu poate continua.

🔒 Cele 4 condiții necesare (Coffman):

  • Mutual Exclusion - resursa poate fi folosită de un singur proces
  • Hold and Wait - procesul ține resurse și așteaptă altele
  • No Preemption - resursele nu pot fi forțat eliberate
  • Circular Wait - lanț circular de așteptare

Gestionarea Memoriei

RAM, memorie virtuală și tehnici de alocare
▼
💾 RAM (Random Access Memory)
De bază

Memoria volatilă principală a calculatorului, unde sunt încărcate programele și datele în timpul execuției. Accesul este rapid dar conținutul se pierde la oprirea alimentării.

📊 Tipuri de RAM:

  • DRAM - Dynamic RAM, necesită refresh periodic
  • SRAM - Static RAM, rapidă, folosită în cache
  • DDR4 - 2133-3200 MHz, standard actual
  • DDR5 - 4800-8400+ MHz, ultima generație
# Linux - informații RAM $ free -h total used free shared buff/cache available Mem: 16Gi 8.2Gi 2.1Gi 512Mi 5.7Gi 7.1Gi # Windows > Get-WmiObject Win32_PhysicalMemory | Select Capacity, Speed
Virtual Memory Cache ROM
🌐 Virtual Memory (Memorie Virtuală)
Avansat

Tehnică de gestionare a memoriei care creează iluzia unei memorii RAM mult mai mari decât cea fizică disponibilă, folosind spațiu pe disc ca extensie.

🔄 Cum funcționează:

  • Paginare - memoria e împărțită în pagini (tipic 4KB)
  • Page Table - mapează adrese virtuale la fizice
  • MMU - hardware care face traducerea
  • Swap/Page File - paginile inactive sunt mutate pe disc
  • Page Fault - când se accesează o pagină lipsă din RAM
RAM Page Fault MMU
⚡ Cache (Memorie Cache)
Avansat

Memorie extrem de rapidă situată între CPU și RAM, care stochează copii ale datelor frecvent accesate pentru a reduce latența accesului la memorie.

📊 Niveluri de cache:

  • L1 - ~32-64KB, cel mai rapid (~1-4 cicluri), per core
  • L2 - ~256KB-1MB, rapid (~10-20 cicluri), per core
  • L3 - ~8-64MB, moderat (~40-75 cicluri), partajat

Sisteme de Fișiere

Organizarea și stocarea datelor pe disc
▼
📂 Sistem de Fișiere (File System)
De bază

Metoda și structura de date folosită pentru a organiza, stoca, găsi și accesa fișierele pe un dispozitiv de stocare.

📦 Sisteme de fișiere populare:

  • NTFS - Windows, jurnalizare, ACL
  • ext4 - Linux, jurnalizare, extents
  • APFS - macOS, criptare, snapshots
  • Btrfs - Linux, COW, snapshots, RAID
  • ZFS - BSD/Linux, RAID-Z, dedup
# Linux - vizualizare sisteme de fișiere montate $ df -Th Filesystem Type Size Used Avail Use% Mounted on /dev/sda1 ext4 500G 120G 380G 24% / # Windows > Get-Volume
Inode Partiție Mount
📋 Inode
Avansat

Structură de date în sistemele de fișiere Unix/Linux care stochează metadatele unui fișier (permisiuni, proprietar, timestamps, locație pe disc) - totul în afară de nume.

📊 Ce conține un inode:

  • Tip fișier (regular, director, link)
  • Permisiuni (rwx pentru user, group, others)
  • UID și GID (proprietar)
  • Dimensiune fișier
  • Timestamps (access, modify, change)
  • Pointeri către blocurile de date
# Vizualizare inode $ ls -li file.txt 123456 -rw-r--r-- 1 user group 1024 Jan 15 10:00 file.txt # Detalii inode $ stat file.txt
File System Hard Link
💽 Partiție
De bază

Secțiune logică a unui disc care poate fi formatată cu un sistem de fișiere propriu. Permite organizarea și izolarea datelor.

📊 Scheme de partiționare:

  • MBR - max 4 partiții primare, 2TB/disc
  • GPT - 128 partiții, 18 EB/disc, modern
File System Mount

Securitate și Permisiuni

Protecția sistemului și controlul accesului
▼
🔑 Permisiuni
De bază

Reguli care definesc ce operații pot fi efectuate asupra unui fișier sau resursă și de către cine. Fundamentale pentru securitatea sistemului.

📋 Sistemul Unix (chmod):

  • r (4) - Read - citire conținut
  • w (2) - Write - modificare conținut
  • x (1) - Execute - rulare ca program
# Linux - vizualizare permisiuni $ ls -l file.txt -rw-r--r-- 1 user group 1024 Jan 15 file.txt # Setare permisiuni $ chmod 755 script.sh # rwxr-xr-x $ chmod u+x script.sh # adaugă execute pentru user
Inode Root
🔐 Mutex (Mutual Exclusion)
Avansat

Obiect de sincronizare care permite doar unui singur thread să acceseze o resursă partajată la un moment dat, prevenind race conditions.

🔄 Operații:

  • lock() - achiziționează mutex-ul (sau așteaptă)
  • unlock() - eliberează mutex-ul
  • try_lock() - încearcă fără blocare
# Python - exemplu mutex import threading counter = 0 mutex = threading.Lock() def increment(): global counter with mutex: # lock() automat counter += 1 # unlock() automat la ieșire
Semaphor Deadlock
⚡ Race Condition
Avansat

Situație în care comportamentul programului depinde de ordinea relativă sau timing-ul evenimentelor necontrolate, ducând la rezultate imprevizibile.

# Exemplu race condition counter = 0 # Thread 1 # Thread 2 read counter (0) read counter (0) increment (1) increment (1) write counter (1) write counter (1) # Rezultat: counter = 1 (ar trebui 2!) # Soluție: folosește mutex
Mutex Thread

Interfețe Utilizator

GUI, CLI și moduri de interacțiune
▼
🖥️ GUI (Graphical User Interface)
De bază

Interfață grafică ce permite utilizatorilor să interacționeze cu sistemul prin elemente vizuale precum ferestre, butoane, meniuri și iconițe, folosind mouse-ul și tastatura.

🖼️ Componente GUI:

  • Window Manager - gestionează ferestrele
  • Desktop Environment - experiență completă (GNOME, KDE)
  • Widgets - elemente UI (butoane, textbox)
CLI Shell
⌨️ CLI (Command Line Interface)
De bază

Interfață text unde utilizatorul introduce comenzi pentru a interacționa cu sistemul. Oferă control precis și posibilitatea de automatizare prin scripturi.

💻 Shell-uri populare:

  • Bash - Bourne Again Shell (Linux default)
  • Zsh - Z Shell (macOS default)
  • PowerShell - Windows, object-oriented
  • CMD - Windows Command Prompt (legacy)
# Comenzi de bază Linux/macOS $ ls -la # listare fișiere $ cd /home # schimbă director $ grep "text" * # caută în fișiere # PowerShell Windows > Get-ChildItem # listare > Set-Location # schimbă director
GUI Shell Terminal
🐚 Shell
De bază

Program care interpretează comenzile utilizatorului și le transmite kernel-ului pentru execuție. Poate fi grafic (Explorer, Finder) sau text (Bash, PowerShell).

🔧 Funcționalități shell:

  • Interpretare și execuție comenzi
  • Variabile de mediu
  • Redirecționare I/O (>, >>, <)
  • Pipe-uri (|)
  • Scripting (automatizare)
CLI Terminal

I/O și Întreruperi

Comunicarea cu dispozitivele și gestionarea evenimentelor
▼
📌 Interrupt (Întrerupere)
De bază

Semnal hardware sau software care întrerupe execuția normală a CPU-ului pentru a trata un eveniment urgent (ex: apăsare tastă, date de la rețea).

📊 Tipuri de întreruperi:

  • Hardware (IRQ) - de la dispozitive externe
  • Software (Trap) - generate de program (ex: syscall)
  • Exception - erori (division by zero, page fault)
  • NMI - Non-Maskable, critice
# Linux - vizualizare întreruperi $ cat /proc/interrupts CPU0 CPU1 0: 45 0 IO-APIC 2-edge timer 1: 523 0 IO-APIC 1-edge i8042
Driver Kernel DMA
📤 I/O (Input/Output)
De bază

Comunicarea între sistem și dispozitivele externe. Input = date primite (tastatură, mouse), Output = date trimise (monitor, imprimantă).

📊 Metode I/O:

  • Polling - CPU verifică repetat starea dispozitivului
  • Interrupt-driven - dispozitivul anunță când e gata
  • DMA - transfer direct în memorie
Driver Interrupt

API și Programare

Interfețe de programare și dezvoltare
▼
🔌 API (Application Programming Interface)
De bază

Set de funcții, protocoale și unelte care permit aplicațiilor să comunice între ele sau cu sistemul de operare într-un mod standardizat.

📦 Tipuri de API:

  • OS API - Win32, POSIX, Cocoa
  • Library API - stdio.h, STL
  • Web API - REST, GraphQL
  • Hardware API - DirectX, OpenGL, Vulkan
System Call POSIX
📋 POSIX
Avansat

Portable Operating System Interface - familie de standarde IEEE care definesc API-ul pentru compatibilitate între sisteme de operare Unix-like.

📦 Ce standardizează POSIX:

  • Apeluri de sistem (open, read, write, fork)
  • Comenzi shell (ls, grep, awk)
  • Threads (pthreads)
  • Expresii regulate
